### **2. Цель проекта**
Ответы на ключевые вопросы инициации.

***Почему создается проект?*** — Для предоставления пользователям доступного инструмента ведения блога без зависимости от зарубежных решений. 

***С какой целью?*** — Создать к 12.12.2025 веб-приложение с базовыми функциями публикации, поиска и фильтрации статей.

***Влияние на стейкхолдеров?*** — Команда получит опыт разработки IT-продукта, заказчик — перспективный продукт для локального рынка.

Таким образом, **цель проекта** — запуск MVP, сочетающего простоту и функциональность.

### **7. Стек технологий**

Для реализации MVP веб-приложения блога выбраны технологии, которые обеспечивают:
- Быструю разработку и простоту сопровождения.
- Масштабируемость и возможность последующего роста проекта.
- Минимальные инфраструктурные затраты.
- Поддержку импортозамещения — возможность развёртывания без критической зависимости от зарубежных сервисов.

<p align="center">
  <strong style="font-size:20px">Основной стек технологий</strong>
</p>

| Компонент | Технология | Обоснование выбора |
|------------|-------------|--------------------|
| Frontend | **Angular + RxJS** | Зрелый, структурированный фреймворк для SPA. RxJS облегчает работу с асинхронными потоками данных, фильтрацией и поиском. |
| API | **REST API** | Простая и гибкая архитектура взаимодействия фронтенда с бэкендом. |
| Backend | **Firebase Functions / Node.js** | Firebase позволяет быстро запустить MVP без собственного сервера. В будущем возможен переход на Node.js для полного контроля. |
| База данных | **Firebase Firestore / PostgreSQL** | Firestore — быстрый прототип, PostgreSQL — надёжная база данных для промышленного использования. |
| Система контроля версий | **Git + GitHub** | Управление кодом, ревью, Pull Request, ведение задач. Возможна миграция на отечественные аналоги. |
| Инфраструктура и деплой | **GitHub Actions / Docker** | Автоматизация CI/CD и контейнеризация для лёгкого развёртывания. |
| Хостинг | **Firebase Hosting*** | Быстрое размещение фронтенда; в будущем можно использовать отечественные облачные сервисы. |
| UI/UX | **Angular Material / Figma** | Быстрое создание адаптивного интерфейса, проектирование UX/UI и документирование пользовательских сценариев. |


#####  Обоснование подхода
- Быстрый запуск MVP без сложной инфраструктуры.
- Низкий порог вхождения для команды.
- Возможность эволюции: Firebase → собственный бэкенд, REST → GraphQL, GitHub → GitLab.
- Поддержка принципов импортозамещения: все ключевые технологии имеют open-source аналоги.

##### Потенциал масштабирования
- Frontend легко расширяется модулями Angular.
- Backend можно перевести на микросервисную архитектуру.
- DevOps: CI/CD с автоматическим тестированием и деплоем.
- Инфраструктура: контейнеризация и оркестрация (Docker / Kubernetes) при росте нагрузки.
